## 二、战斗系统实现路线图

### 阶段 1：核心回合制逻辑（优先级：最高）

目标：实现可玩的基础战斗循环。

- [x] **行动顺序系统**：基于速度(Speed)属性排列行动队列 — 见 [CombatTurnSystem.cs](../Assets/Scripts/Pal3.Game/GameSystems/Combat/CombatTurnSystem.cs)
- [x] **回合状态机**：Idle → TurnStart → ActorActing → TurnEnd → CheckResult → Finished — 见 [CombatPhase.cs](../Assets/Scripts/Pal3.Game/GameSystems/Combat/CombatPhase.cs)
  - 已替换 `CombatManager.Update()` 中的测试按键逻辑
- [x] **伤害计算**：基础公式 `Attack - Defense/2`，含暴击和小幅随机浮动 — 见 [DamageCalculator.cs](../Assets/Scripts/Pal3.Game/GameSystems/Combat/DamageCalculator.cs)。元素克制和伤害公式精修留到阶段2/3
- [x] **运行时状态**：HP/SP/MP 实时变化、死亡判定 — 见 [CombatActorRuntimeState.cs](../Assets/Scripts/Pal3.Game/GameSystems/Combat/CombatActorRuntimeState.cs)
- [x] **被攻击/死亡动画**：`CombatActorController.ApplyDamageAsync()` 触发 BeAttack / Dead 动作
- [x] **战斗结果判定**：任一方全灭即结束，结果通过 `OnCombatFinished` 事件分发

#### 阶段1 已知限制（留待后续阶段）
- 玩家行动选择仍是自动随机选目标，无 UI 菜单（属于阶段4）**已实现**
- `AnimationEventTriggered` 中的 work/flydown 钩子仍未对接到伤害结算时机
- 偷袭/被偷袭场景未让先手方全员先行动（目前只按速度排序）
- 元素克制、状态效果（毒/麻痹/混乱等）未实现（属于阶段2/3）

## 五、开发原则

1. **遵循现有架构**：所有新系统通过 Command 驱动，在 `Pal3.cs` 中注册，不引入新的架构模式
2. **数据驱动**：战斗数值、商品数据从原版数据文件读取，不硬编码
3. **分阶段可测试**：每个阶段完成后应有可运行的测试场景
4. **文档同步**：每完成一个阶段，更新本文档状态
